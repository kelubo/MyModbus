## 🔧 配置文件

### 环境变量 (.env)
```bash
PORT=3000                    # 服务器端口
SYSTEM_NAME=我的监控系统      # 系统名称
SYSTEM_LOGO=/images/logo.svg # Logo 路径
TZ=Asia/Shanghai             # 时区
```

### 数据库配置 (config/database.config.js)
```javascript
module.exports = {
  type: 'sqlite',           // sqlite/mysql/postgresql
  sqlite: {
    filename: './modbus.db'
  }
};
```

### 集群配置 (config/cluster.config.js)
```javascript
module.exports = {
  enabled: false,           // 启用集群
  node: {
    id: 'node-1',          // 节点 ID
    role: 'both'           // master/worker/both
  }
};
```

## 📊 数据类型

### Modbus 数据类型
| 类型 | 说明 | 读取方式 |
|------|------|----------|
| holding | 保持寄存器 | 读写 |
| input | 输入寄存器 | 只读 |
| coil | 线圈 | 读写 |
| discrete | 离散输入 | 只读 |

### 连接类型
| 类型 | 说明 | 参数 |
|------|------|------|
| RTU | 串口连接 | 串口、波特率 |
| TCP | 网络连接 | IP地址、端口 |

## 🔍 故障排除

### 常见问题

**Q: 无法连接设备？**
A: 检查串口/IP地址、波特率/端口、从站ID是否正确

**Q: 数据不更新？**
A: 确认已启动数据采集，检查 WebSocket 连接

**Q: 告警未触发？**
A: 检查告警规则是否启用，阈值设置是否正确

**Q: 通知未收到？**
A: 检查通知配置，使用测试功能验证

**Q: 页面显示异常？**
A: 清除浏览器缓存，硬刷新（Ctrl+F5）
